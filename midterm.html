Seatworks

Seatwork 1

This seatwork tested our knowledge of unary operators. It consisted of multiple lines of tricky unary operator equations and we had to determine if it was TRUE or FALSE, and what the computed answer was (if it was an arithmetic equation). I found it confusing but doable. I had a great start, but I lost track of which line I was answering. With the help of my seatmates, I managed to catch up and finish the seatwork. I loved how we helped each other while still making sure that we were learning ourselves.

Seatwork 2

This seatwork required us to create a system that functions like an ATM machine. Provide a menu, let users decide which action to do (out of 4 available), and count the total transactions. Through this seatwork, I was able to utilize the different functions and operators of JAVA; scanner import, while loops, and switch case in building my program. I especially practiced my while loop logic through this activity as I was still a noob at the time. In the end, my program turned out exactly how I wanted it to be and it hit all of the requirements of the seatwork.

Seatwork 3

This seatwork required us to utilize the 'method' feature of JAVA to create a system that can analyze students' age. Let the user input their name and age, and pass that information to another method to analyze which category they belong to. This was a relatively easy activity for me as the requirements were simple. I just made two methods: one was the main and the other was to evaluate and categorize the students' age. However, I failed to realize that I passed the wrong output. Instead of passing a scanned version of the handwritten code, I passed a pdf version of my screenshots of the output and code. Hence, my submission was deemed invalid. Until now, I am still waiting for my professor's verdict (my score t_t).

Activities

Activity 1

This activity tested our knowledge about JAVA's variables. It had three parts: Part A, identify each variable's data type and where they're stored in; Part B, explain what happens to the given line of code and its memory impact; and Part C, evaluate what output the given code will print. This helped strengthen my foundation and familiarity to JAVA's different variable types, memory systems, as well as the syntax because of Part C where we evaluated lines of JAVA code.

Activity 2

This activity tested our knowledge about JAVA's operators. It had two parts, both of which required us to use different operators in our program (such as bitwise, ternary, assignment, and arithmetic). I utilized most of them in my program and achieved all of the requirements. However, in Part II, I went a little overboard. I created a program that computes a student's grade and identifies if they passed or not. Yes, it did have the required operators but I missed a few. I forgot to put a tricky increment or decrement case, lacked 3 numeric results, and my implementation went beyond the required tasks. I only realized it after our scores were returned. Back then, I wanted to implement the program in a way that is somewhat realistic, but in doing so, I overlooked a few details. Looking back, I think my idea for the program could have been executed in a better way. Realistic, while still maintaining the required items.

Activity 3

This activity is the complicated version of the Basic ATM System seatwork. Continuously ask for user input using a while loop, proceed with what the user wants using a switch case, call the respective function to perform the task, and count only the successful transactions. I had quite the fun yet stressful time building this program. I liked organizing each option and coding their functions, but it was stressful making sure that you've thought of every scenario and every possible errors. Previosly, I practiced my while loop skills on the Basic ATM System seatwork and while doing this activity, I found that I've improved! In the end, I achieved every required items in my program.

Activity 4

This activity required us to evaluate a student's grade. It's similar to what I built in Activity #2 - Part II, only this time the input is hardcoded. So I only needed to build the grade evaluator and output the answer. I used arithmetic operators to calculate the average and a multi-line if-else statement to determine the grade remarks. Finally, I passed my program with complete requirements.

Activity 5

Looking back at everything I've done for this midterm, I can honestly say that I learned a lot, more than I expected at the beginning of the semester. At first, Object-Oriented Programming felt a bit intimidating, especially since JAVA was still new to me and because from what I kept on seeing from my seniors, it had a lot of syntax or long words. But as I went through each quiz, seatwork, and activity, I slowly started to understand how things worked.

One thing I noticed while reflecting on my outputs is that my performance really depended on how prepared I was. For example, I did well in Quiz 1 because I reviewed beforehand, while Quiz 2 didn’t go as smoothly since I didn’t prepare enough. The same pattern shows in some of my activities and seatworks. That honestly hit me because I knew I could have done better if I managed my time more seriously.

Despite that, I genuinely enjoyed working on the programming tasks. Activities like the ATM System and Simple Wallet System were stressful, but in a good way. I liked figuring out the logic, organizing the flow, and seeing my program finally work the way I intended. It made me realize that JAVA is actually an enjoyable language when you put in the effort to learn it!

Also, I absolutely cannot NOT mention just how fun the face-to-face class itself was. There were moments that made the class memorable, like when our Prof. Bedis gave us Star City coupons (even though they expired that Sunday!!).

Overall, this midterm experience taught me a lot, not just academically but also personally. I realized hard that I really need to step up my time management game and that preparation is of utmost importance when you want to achieve the best results. I know it's an obvious thing and I'm aware of it, but maybe I just needed to realize that again so I can improve myself more.

I'm hoping for more face-to-face classes this semester !
